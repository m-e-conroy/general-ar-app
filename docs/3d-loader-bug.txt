PROBLEM:
The GLTF model was compressed using Draco compression (a geometry compression format).
GLTFLoader requires a DRACOLoader instance to decompress Draco-compressed models.
Without it, the loader cannot parse the compressed geometry data.

SOLUTION:
1. Imported DRACOLoader from 'three/addons/loaders/DRACOLoader.js'
2. Created a DRACOLoader instance in the loadGLTFModel() method
3. Configured it to use Google's hosted Draco decoder library (v1.5.6)
4. Attached the DRACOLoader to the GLTFLoader using setDRACOLoader()

The model should now load successfully. Draco compression is commonly used to
reduce file sizes for 3D models, especially for web delivery.

- main.js: Added DRACOLoader import
- main.js: Added DRACOLoader setup in loadGLTFModel() method
